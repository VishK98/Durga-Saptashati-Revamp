1. Essential Meta Tags From Old Project

These meta tags must be present in the <head> of every page:

<!-- Character Set & Viewport -->
<meta charset="UTF-8">
<meta http-equiv="X-UA-Compatible" content="IE=edge">
<meta name="viewport" content="width=device-width, initial-scale=1.0">

<!-- SEO Core -->
<meta name="robots" content="index, follow">
<title>Your Page Title — Saptashati Foundation</title>
<meta name="description" content="Page-specific description, 150-160 characters max.">
<meta name="keywords" content="relevant, comma, separated, keywords">

<!-- Language -->
<html dir="ltr" lang="en">
Tip: Keep the title under 60 characters and the description under 160 characters for best SERP display. The old project had excessively long keyword tags — trim to 10-15 highly relevant keywords per page.

2. Canonical URLs From Old Project

Add one canonical tag per page pointing to the full, canonical URL of that page:

<!-- Homepage -->
<link rel="canonical" href="https://www.saptashati.org/">

<!-- Inner Pages -->
<link rel="canonical" href="https://www.saptashati.org/about.php">
<link rel="canonical" href="https://www.saptashati.org/contact.php">
<link rel="canonical" href="https://www.saptashati.org/make-donation.php">
Issue in Old Project: Several event pages had canonical tags pointing to wrong URLs (e.g., multiple pages pointing to international-womens-day.html). Ensure every page's canonical matches its own URL exactly.

These were completely absent in the old project. Add them to every page for proper social media sharing:

8. .htaccess — Redirects & Rules From Old Project

File: public/.htaccess

# Custom 404 Error Page
ErrorDocument 404 /404.php

<IfModule mod_rewrite.c>
  RewriteEngine On

  # ============================================================
  # HTTPS Enforcement (HTTP → HTTPS)
  # ============================================================
  RewriteCond %{HTTPS} off
  RewriteRule (.*) https://%{HTTP_HOST}%{REQUEST_URI} [L,R=301]

  # ============================================================
  # WWW Enforcement (non-www → www)
  # ============================================================
  RewriteCond %{HTTP_HOST} !^www\. [NC]
  RewriteRule (.*) https://www.%{HTTP_HOST}%{REQUEST_URI} [L,R=301]

  # ============================================================
  # Remove index.php from URL
  # ============================================================
  RewriteCond %{THE_REQUEST} /index\.php? [NC]
  RewriteRule ^(.*/)?index\.php?$ /$1 [R=301,L]

  # ============================================================
  # 301 Redirects: Old URLs (.html) → New URLs (.php)
  # Preserves SEO juice from the old site
  # ============================================================
  RewriteRule ^index\.html$                    / [R=301,L]
  RewriteRule ^about\.html$                    /about.php [R=301,L]
  RewriteRule ^contact\.html$                  /contact.php [R=301,L]

11. Image SEO & Alt Text From Old Project

Follow these rules for every image on the new site:

<!-- GOOD: Descriptive alt text -->
<img src="assets/images/events/independence-day-2025.jpg"
     alt="76th Independence Day celebration at Saptashati Foundation, Dwarka"
     loading="lazy"
     width="800"
     height="450">

<!-- GOOD: Logo with simple alt -->
<img src="assets/images/logo-wide.png"
     alt="Saptashati Foundation Logo"
     width="200"
     height="60">

<!-- BAD: Avoid these patterns -->
<img src="image.jpg" alt="logo">         <!-- Too vague -->
<img src="image.jpg" alt="">              <!-- Empty alt -->
<img src="image.jpg">                     <!-- Missing alt -->

Image SEO Rules

  • Every <img> must have an alt attribute
  • Use descriptive, keyword-rich alt text (but don't keyword-stuff)
  • Add loading="lazy" to all below-the-fold images
  • Always specify width and height to prevent layout shift (CLS)
  • Use WebP format where possible for smaller file sizes

14. Final Checklist

Per-Page Checklist

  • Unique <title> tag (under 60 characters)
  • Unique meta description (under 160 characters)
  • Correct canonical URL matching the page's own URL
  • Open Graph tags (og:title, og:description, og:image, og:url)
  • Twitter Card tags
  • JSON-LD Organization schema
  • All images have descriptive alt text
  • Images have width, height, and loading="lazy"
  • Only one <h1> tag per page
  • Semantic heading hierarchy (h1 → h2 → h3)

Site-Wide Checklist

  • robots.txt in public root with sitemap reference
  • sitemap.xml listing all public pages
  • .htaccess with HTTPS + WWW enforcement
  • .htaccess with .html → .php 301 redirects (old site migration)
  • Google Search Console verification files copied
  • Google Analytics 4 tracking code on all pages
  • Favicon + Apple Touch Icon in place
  • OG default image created (1200x630px)
  • All internal links use HTTPS and consistent URLs
  • No broken links (run a crawler to check)
  • Submit new sitemap to Google Search Console
